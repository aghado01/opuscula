# opuscula

A small collection of *opuscula* — minor works — shared publicly for review. Each is a
snapshot-in-time vignette of a larger private project, curated down to what stands on its own.

Two here, and they're a pair:

- **[codex-membrane](codex-membrane/)** — a pure-PowerShell MCP server that repairs machine-extracted
  scientific PDFs into faithful, LaTeX-consistent markdown. An agentic apparatus: a deterministic
  preprocess pipeline grades the damage, and a seeing agent with its sub-agent swarm resolves what's
  left against an audited work-list.
- **[codex-scientiae](codex-scientiae/)** — a sample of the bound-knowledge corpus that pipeline
  produces: long-form textbooks (*codices*) and theme-ordered paper collections (*compendia*),
  structured for agent retrieval and navigated by telescoping `CONTENTS` files.

One binds knowledge; one is the knowledge store. Both are built agent-first — to be operated and read
by tool-using models, not only by people.

> Snapshots, not products. Curated subsets of active private repositories — under development, not
> production-certified. Each opusculum carries its own `README` and `AGENTS.md`.
